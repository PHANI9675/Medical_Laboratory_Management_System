2. Remove /h2-console/**, /dev/**, and /test-ui.html from SecurityConfig.java (basically lines marked with "// <-- remove this line before integration"


3. Restore application.yml — re-enable Eureka and Config Server


From this:

# Eureka — set enabled=false so startup does NOT fail if Eureka is not running
eureka:
  client:
    enabled: false    # ← restore to true before final integration
    register-with-eureka: false   # ← restore to true before final integration
    fetch-registry: false   # ← restore to true before final integration
    service-url:
      defaultZone: http://localhost:8761/eureka/
  instance:
    prefer-ip-address: true


to this:


# Eureka — set enabled=false so startup does NOT fail if Eureka is not running
eureka:
  client:
    enabled: true
    register-with-eureka: true
    fetch-registry: true
    service-url:
      defaultZone: http://localhost:8761/eureka/
  instance:
    prefer-ip-address: true



4. Remove the direct Feign URL overrides from application.yml


From this:

# Feign client timeouts + standalone URL overrides
# When running without Eureka, Feign uses these URLs directly
# instead of trying to resolve via load balancer

  client:
    config:
      default:
        connect-timeout: 5000
        read-timeout: 5000
      inventory-service:      # ← REMOVE this block before final integration
        url: http://localhost:8084
      user-service:           # ← REMOVE this block before final integration
        url: http://localhost:8081


to this:

# Feign client timeouts + standalone URL overrides
# When running without Eureka, Feign uses these URLs directly
# instead of trying to resolve via load balancer
feign:
  client:
    config:
      default:
        connect-timeout: 5000
